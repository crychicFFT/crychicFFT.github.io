CAN 总线通信详细页

CAN 是工业、汽车和机器人领域常见的高可靠总线。它允许多个节点共享一条总线，通过报文 ID 仲裁决定谁先发送，并具备较强的错误检测能力。

核心概念 总线结构 仲裁动画 显性/隐性位 报文结构 工程参数 排错清单

核心概念

可以把 CAN 理解为“多设备共享的广播总线”：节点不是直接给某个设备发消息，而是在总线上广播带 ID 的报文。

Signal CAN_H / CAN_L

使用差分信号传输，适合强干扰环境。总线两端通常需要终端电阻。

Message 基于报文 ID

CAN 关注“这条消息是什么”，而不是“发给哪个地址”。节点根据 ID 决定是否处理。

Arbitration ID 越小优先级越高

多个节点同时发送时，会在仲裁阶段自动决定优先级，不会像普通串口那样直接冲突。

总线结构动画

CAN 通常使用一对 CAN_H / CAN_L 差分线。多个节点挂在总线上，最两端加终端电阻。

接线重点

真实项目里，CAN 调不通经常不是协议问题，而是总线接线、终端电阻、波特率或 CAN_H / CAN_L 接反。

CAN_H差分高线，与 CAN_L 配合传输总线状态。
CAN_L差分低线，与 CAN_H 共同形成差分电压。
120Ω总线两端常用 120Ω 终端电阻，减少信号反射。
GND有些场景建议连接参考地，提升系统稳定性。

仲裁动画

CAN 的经典特点是“无损仲裁”：多个节点同时发送时，优先级低的节点会主动退出，高优先级报文继续发送。

仲裁结果：0x120 优先级更高，Node A 继续发送；Node B 暂停并等待下一次机会。

为什么 ID 小优先？

CAN 仲裁利用显性位和隐性位的关系。显性位会覆盖隐性位。节点一边发送一边监听总线，如果发现自己发送的是隐性位，但总线上是显性位，就说明有更高优先级报文，自己退出。

ID 小在二进制比较中更早出现显性 0，因此优先级更高。
不破坏低优先级节点退出，高优先级报文不会被破坏。
适合实时重要消息可以设计更小 ID，确保更快获得总线。

显性位与隐性位

CAN 的仲裁和错误处理都和显性/隐性位有关。直观理解：显性位更“强”，隐性位更“弱”。

隐性位 1

显性位 0

直观理解： 隐性状态下 CAN_H 与 CAN_L 接近；显性状态下两者产生明显差分电压。

电平逻辑

CAN 是差分总线，接收器判断 CAN_H 和 CAN_L 的电压差。显性位通常表示逻辑 0，隐性位通常表示逻辑 1。

隐性 1总线没有节点主动拉出显性差分，CAN_H 与 CAN_L 较接近。
显性 0节点驱动总线产生差分电压，覆盖隐性状态。
仲裁如果你发 1 但读到 0，就说明别人优先级更高。

工程参数与应用

CAN 在机械臂、底盘、电机驱动器、汽车电子中非常常见。工程上重点关注波特率、终端电阻、ID 规划和错误状态。

项目 常见情况 说明 错误表现
波特率 125k / 250k / 500k / 1M 同一条总线上的节点必须一致。 完全无法通信或大量错误帧。
终端电阻 总线两端 120Ω 不是每个节点都加，而是总线两端加。 偶发掉线、远距离不稳定。
CAN_H / CAN_L 差分线 接反通常无法通信，部分设备可能有保护。 总线无数据或错误计数上升。
报文 ID 标准 11 位 / 扩展 29 位 ID 既表示消息类型，也影响优先级。 消息被过滤、解析不到目标数据。
应用场景 电机、底盘、BMS、机械臂 适合多节点、高可靠、实时控制场景。 控制响应异常或状态反馈缺失。

常见排错清单

CAN 调不通时，可以按“物理层 → 波特率 → ID → 协议解析 → 错误状态”顺序排查。

硬件排查

01确认 CAN_H 和 CAN_L 是否接反。
02确认总线两端是否有 120Ω 终端电阻。
03确认不是星形乱接，尽量保持主干总线结构。
04确认 USB-CAN、CAN 收发器、供电和地线是否正常。

软件排查

01确认所有节点波特率一致。
02确认标准帧/扩展帧设置一致。
03确认报文 ID 是否被过滤器过滤。
04查看错误计数、Bus Off 状态和驱动日志。
