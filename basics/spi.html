SPI 通信详细页

SPI 是常见的高速同步通信接口，通常由主机产生时钟，通过 MOSI 向从机发送数据，同时通过 MISO 从从机读取数据。它速度快、逻辑直接，但占用引脚比 I2C 更多。

核心概念 总线结构 时序动画 字段与模式 参数与应用 排错清单

核心概念

SPI 的重点是“主机控制时钟、片选选择从机、MOSI/MISO 同时传输”。

Mode 同步通信

主机通过 SCLK 提供时钟，从机按时钟节拍发送或接收数据。

Wires SCLK / MOSI / MISO / CS

常见四类信号，多个从机通常需要多个片选 CS。

Feature 高速全双工

主机发送数据的同时，也可以从从机接收数据。

总线结构动画

SPI 通常由一个主机控制多个从机。SCLK、MOSI、MISO 可以共享，CS 用来选择当前通信的从机。

接线重点

SPI 比 I2C 更快，但线更多。一个主机挂多个从机时，每个从机通常需要独立 CS 片选线。

时序动画

SPI 通过 SCLK 边沿采样数据。MOSI 和 MISO 可以在同一组时钟下同时传输。

CS SCLK MOSI MISO

怎么看 SPI 时序？

先看 CS 是否有效，再看 SCLK 的极性与相位，最后看 MOSI/MISO 数据在哪个边沿被采样。不同设备手册会规定 SPI Mode 0/1/2/3。

CS 有效从机被选中后才响应 SCLK 和数据。
SCLK主机输出时钟，从机按指定边沿采样数据。
全双工主机每发出一个 bit，通常也会同时读回一个 bit。

参数与应用

SPI 常用于高速板级外设，但线数较多，不适合长距离通信。

项目 常见情况 说明 错误表现
线数SCLK / MOSI / MISO / CS基础四线，多个从机需要更多 CS。片选错会读不到目标设备。
速度MHz 级常见通常比 I2C 快，具体取决于芯片和布线。速度太高会导致数据错位或读写失败。
模式Mode 0 / 1 / 2 / 3由 CPOL 和 CPHA 组合决定。模式不匹配会读到错误数据。
位序MSB first / LSB first多数设备默认 MSB first。字节位序异常，解析错误。
应用Flash、屏幕、ADC、无线模块适合高速、短距离、板级外设通信。长线时容易受干扰。

常见排错清单

SPI 调不通时，优先确认 CS、模式、时钟速度、线序和芯片手册中的读写时序。

硬件排查

01确认 SCLK、MOSI、MISO、CS 是否接对。
02确认主从设备电平是否一致，例如 3.3V 与 5V。
03确认 CS 是否真的拉到有效电平。
04降低时钟速度测试，排除布线和信号完整性问题。

软件排查

01确认 SPI Mode 是否和芯片手册一致。
02确认 MSB/LSB 位序设置是否正确。
03确认读操作前是否需要先发送命令字或寄存器地址。
04确认每次传输前后 CS 的拉低和释放时序。
