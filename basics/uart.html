UART 串口通信详细页

UART 是最常见的基础通信接口之一。它适合点对点、低到中速、调试友好的设备通信。这个页面先建立完整框架：概念、引脚、帧结构、接线方式、动画演示和常见排错。

核心概念 引脚说明 帧结构动画 接线示意 常见参数 排错清单

核心概念

先把 UART 看成两台设备之间的一条“文字通道”：一端 TX 发，一端 RX 收，双方提前约定通信速度。

Mode 异步通信

没有单独时钟线，发送端和接收端通过相同波特率来理解数据位。

Wires TX / RX / GND

基础通信至少需要发送线、接收线和共地。很多调试串口就是这三根线。

Frame 起始位 + 数据位 + 停止位

接收端通过起始位发现一帧开始，然后按约定节奏采样数据位。

引脚说明

UART 最容易出错的地方是 TX/RX 接反、没有共地、波特率不一致。

常见引脚

不同开发板、USB 转串口模块、传感器模块的标注可能不同，但核心逻辑相同。

TXTransmit，发送端输出。连接到对方的 RX。
RXReceive，接收端输入。连接到对方的 TX。
GND公共参考地。没有共地时，电平参考可能不一致。
VCC电源脚，不一定需要接。调试时要确认电压是否匹配。

接线原则

UART 是“交叉接线”：A 的 TX 接 B 的 RX，A 的 RX 接 B 的 TX。GND 要连接在一起。

1不要把 TX 接 TX、RX 接 RX，除非设备文档明确做了内部转换。
2注意 3.3V 与 5V 电平差异，必要时使用电平转换。
3常见配置格式是 115200-8-N-1：波特率 115200，8 位数据，无校验，1 位停止位。

起始位

空闲状态通常为高电平。当发送端拉低电平时，接收端认为一帧数据开始。

理解重点： UART 没有时钟线，所以接收端必须依赖“起始位 + 波特率”来判断后续每一位的位置。

接线示意

真实项目中，先确认电平、电源、共地，再确认 TX/RX 是否交叉。

MCU / PC控制端
Sensor外设模块
TX → RX
RX ← TX
GND ↔ GND
UART 基础接线：交叉收发 + 共地。VCC 是否连接取决于模块供电方式。

常见参数

串口能否正常通信，经常取决于两端参数是否一致。

参数 常见值 说明 错误表现
波特率 9600 / 115200 / 1Mbps 每秒传输符号数，双方必须一致。 乱码、收不到完整数据。
数据位 8 bit 一帧中真正承载数据的位数。 字节解析异常。
校验位 None / Even / Odd 用于基础错误检测。 接收报错或数据被丢弃。
停止位 1 / 2 bit 表示一帧结束。 帧错误、通信不稳定。
电平 3.3V / 5V / TTL 不同设备电平要兼容。 无法识别，甚至损坏 IO。

常见排错清单

串口调不通时，不要一开始怀疑代码，先按硬件和参数逐项排查。

硬件排查

01确认 TX/RX 是否交叉连接。
02确认两端是否共地。
03确认 3.3V / 5V 电平是否兼容。
04确认 USB 转串口芯片驱动是否正常。

软件排查

01确认波特率、数据位、校验位、停止位一致。
02确认串口号是否选对，例如 Linux 下的 /dev/ttyUSB0。
03确认程序是否有串口权限。
04用串口助手先测试，再写复杂程序。
