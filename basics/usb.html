USB 通信详细页

USB 是上位机连接外设最常见的接口之一。它不仅是一根线缆，也包含主机、设备、枚举、端点、设备类和驱动模型。很多常见工具本质上都是 USB 转其他通信接口。

核心概念 连接结构 枚举流程 设备类 转接场景 排错清单

核心概念

USB 的重点是“主机管理设备”：设备插入后，主机会识别设备类型、分配地址、加载驱动，然后建立数据通道。

Role Host / Device

USB 通常由主机发起和管理通信，外设作为 Device 响应主机请求。

Signals D+ / D- / VBUS / GND

D+ 和 D- 用于差分数据传输，VBUS 提供电源，GND 提供参考地。

Model 端点 / 设备类 / 驱动

应用层通常通过串口、HID、批量传输或厂家驱动访问 USB 设备。

接线与角色

USB 枚举流程

设备插入电脑后，系统并不是立刻“直接通信”，而是先完成枚举和驱动匹配。

1. 插入设备主机检测到 USB 设备接入，开始复位和初始化。
2. 读取描述符主机读取设备描述符、配置描述符、接口描述符等信息。
3. 分配地址主机给设备分配 USB 地址，后续通过该地址访问设备。
4. 匹配驱动系统根据 VID、PID、设备类匹配驱动程序。
5. 建立通道应用程序通过串口、HID、Bulk 等方式访问设备。

参数与应用

USB 的复杂度比普通串口更高，但生态成熟，适合电脑连接各类外设。

项目 常见情况 说明 错误表现
角色Host / Device主机管理设备，设备响应主机请求。两个 Device 直接相连通常不能通信。
识别VID / PID / 设备类系统根据这些信息加载驱动。设备未知、驱动异常、无法打开。
端点Endpoint 0 / IN / OUTUSB 数据传输通过端点完成。应用程序找不到正确通道。

带宽取决于 USB 版本和设备视频、相机、大量数据会占用较高带宽。丢帧、卡顿、设备断连。

常见排错清单

USB 调不通时，先看系统是否识别，再看驱动和权限，最后再看具体协议。

硬件排查

01换 USB 线，确认这根线支持数据传输，不只是充电线。
02确认设备供电是否足够，尤其是相机、采集卡、开发板。
03换 USB 口或避免劣质 Hub，排除接触不良和带宽不足。
04确认转接器设备侧接线正确，例如 UART、RS485、CAN。

软件排查

01Windows 看设备管理器，Linux 看 lsusb、dmesg、/dev/ttyUSB*。
02确认驱动是否安装正确，例如 CH340、CP210x、FTDI、USB-CAN 驱动。
03Linux 下确认串口权限，例如 dialout 组或临时 chmod。
04确认应用程序没有重复占用同一个 USB 设备或串口。
