I2C 通信详细页

I2C 是常见的板级短距离通信接口，只需要 SCL 和 SDA 两根信号线，就可以让一个主机连接多个从机。它常用于传感器、EEPROM、OLED、IMU 等模块。

核心概念 总线结构 Start / Stop 时序字段 参数与应用 排错清单

核心概念

I2C 的重点是“两根线、多设备、地址寻址、主机发起”。它适合板子内部或短距离模块通信。

Wires SCL / SDA

SCL 是时钟线，SDA 是数据线。数据在时钟节拍下传输。

Topology 一主多从

主机控制通信开始和时钟，从机通过地址被选中。

Electrical 开漏 + 上拉

SCL/SDA 通常需要上拉电阻，设备通过拉低总线来表达低电平。

总线结构动画

I2C 通过 SCL 和 SDA 两根线连接多个设备。主机发出地址，目标从机应答 ACK。

接线重点

I2C 看起来线少，但调试时很依赖上拉电阻、地址是否冲突、线长和时钟速度。

SCLSerial Clock，时钟线，由主机产生节拍。
SDASerial Data，数据线，地址、读写位、数据和 ACK 都走这根线。
上拉I2C 常用开漏输出，SCL/SDA 需要通过电阻拉到高电平。
地址每个从机有地址，主机通过地址选择目标设备。

Start / Stop 条件

I2C 通过 SDA 在 SCL 为高时的跳变来表示通信开始或结束。

SCL SDA
Start：SDA ↓ while SCL high
Stop：SDA ↑ while SCL high

如何判断一帧开始？

当 SCL 保持高电平时，SDA 从高变低，表示 Start 条件；当 SCL 保持高电平时，SDA 从低变高，表示 Stop 条件。

Start主机宣布：我要开始一次通信。
Stop主机宣布：本次通信结束，释放总线。
Repeated有些读操作会使用重复起始条件，不立即释放总线。

参数与应用

I2C 常用于板级外设。它线少、方便，但不适合长距离和强干扰环境。

项目 常见情况 说明 错误表现
线数 SCL / SDA / GND 两根信号线即可挂多个设备。 少接 GND 时通信不稳定。
速度 100k / 400k / 1M 常见标准模式 100k，快速模式 400k。 速度太高会导致 ACK 丢失或读写失败。
上拉电阻 常见 2.2kΩ ~ 10kΩ 与总线电容、速度、电压有关。 波形上升慢、设备无响应。
地址 7 位地址常见 同一总线上从机地址不能冲突。 读到错误设备或总线冲突。
应用 IMU、OLED、EEPROM、传感器 适合短距离、低到中速、板内模块通信。 长线时容易受干扰。

常见排错清单

I2C 调不通时，最先看地址、上拉电阻、电平、接线和示波器波形。

硬件排查

01确认 SCL、SDA 是否接反。
02确认 SCL/SDA 是否有合适上拉电阻。
03确认电平是否匹配，例如 3.3V 和 5V 是否需要电平转换。
04确认线长不要过长，总线电容不要过大。

软件排查

01确认从机地址是否正确，注意 7 位地址和 8 位地址写法差异。
02确认读写寄存器地址是否正确。
03确认设备是否需要先写寄存器地址再读数据。
04降低 I2C 速度测试，例如先用 100kHz。

继续学习

I2C 适合线少、板内多外设；如果追求更高速度，可以继续学习 SPI。
